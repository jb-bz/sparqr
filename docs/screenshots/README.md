## How these are generated

The PNGs are rendered by `bin/render-kanban.py` (see `/tmp/render-kanban.py`
in the dev session) which:

1. Takes a title and a `.txt` file (raw terminal output) as input
2. Builds a small HTML page styled to look like a dark-themed terminal
3. Renders it via headless Chrome (`--headless --screenshot=...`)
4. Saves a 1100x800 PNG

The result looks like a real terminal screenshot but is generated
from text — so the content is always accurate (no out-of-date
screenshots) and small (~30-80KB per PNG).

## Adding more screenshots

```bash
# 1. Capture the actual command output
sparc new my-app --type web-app > screenshots/my-command.txt 2>&1

# 2. Render it as a PNG
python3 bin/render-kanban.py "My command" screenshots/my-command.txt screenshots/my-command.png

# 3. Reference it in docs:
# ![My command](docs/screenshots/my-command.png)
```

## Why no real Hermes dashboard screenshots?

The Hermes webui (started via `hermes dashboard --port 8787`) renders
a React SPA that connects to a backend gateway. The frontend requires
a desktop-app session token, so headless browsers can't get past
the auth screen. The kanban-board-as-text screenshots above are a
faithful substitute and don't require running infrastructure.
