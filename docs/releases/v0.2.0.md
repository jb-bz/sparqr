# sparqr v0.2.0 — "Make it work reliably"

**40 of 40 story points done. 9 of 9 stories shipped. 237 unit tests pass.**

The biggest release yet for sparqr. v0.1.0 was a working prototype; v0.2.0 is something you can actually rely on for daily work. This release cuts orchestration latency by 12x, makes the package recoverable when agents crash, adds structured reviewer workflows, and gives you the tools to optimize cost per stage.

## What's new

### 🚀 Event-driven polling (250ms instead of 3s)

The orchestrator now responds to kanban changes in 250ms instead of waiting up to 3 seconds for the next poll. Uses Hermes Kanban's existing `task_events` SQLite table — no sidecar DB, no extra infrastructure.

### 🔧 Stale-task reaper

Agent crashes (OOM, Ctrl-C, network blip) used to leave tasks in `running` state forever, silently stalling the pipeline. The reaper detects dead agent PIDs and re-queues the task as `ready` with a `[REAPED attempt N/M ...]` comment. After `max_attempts` (default 2) reaps without progress, blocks the task for human intervention. **9-minute deploys are safe** — the PID is alive, the reaper doesn't fire.

### 📋 Reviewer checklist skill

The reviewer agent now has structured guidance: read the artifact, extract acceptance criteria from the spec, post a markdown review with PASS/FAIL per criterion and an explicit verdict (APPROVE / REJECT / REDIRECT), then `kanban_block`. The HITL adapter surfaces the structured review to whatever UI you configured.

### 💰 Per-stage model routing

`sparc.config.yaml` gains a `models:` section. Route cheap stages (spec, design, pseudocode) to a faster model; expensive stages (architecture, refinement, completion) to a stronger one. Opt-in — if you don't set it, behavior is unchanged.

### 🧪 Integration test framework

VCR-style record-replay harness for testing against real Hermes. First run captures real Hermes output to JSON fixtures; subsequent runs replay them without Docker. Framework ships with Docker compose for real-Hermes recording and CI integration (runs on main merges only, not PRs).

### ⚙️ CI workflow + prerequisites check

GitHub Actions with shellcheck + full test suite on every PR. New `sparc-doctor --pre-install` (and `setup.sh` itself) checks for bash ≥4.0, hermes, sqlite3, curl, jq before doing anything destructive.

## Quality improvements

- **Function-hoisting bug in bash** caught by `test_e2e.sh` before shipping — function definitions were below the case statement that called them
- **Pre-existing preflight test env-dependence** fixed by stripping PATH inside the test only (deterministic now)
- **Hermes version compatibility documented** — `lib/kanban.sh` header records tested-against, minimum compatible, and the one known quirk
- **All estimates reformatted from human-time to Fibonacci story points** per user feedback — "no human time, only story points"

## Known limitations (carried forward)

- **Story 6 (integration tests) scaffolded but not fully recorded**: the framework works, but no real Hermes sessions are recorded yet. Recording requires the official Hermes Docker image, which isn't published yet. v0.2.1 task once the image is available.
- **Reaper is PID-only**: doesn't catch alive-but-stuck agents. Time-based fallback is a v0.3.0 story.
- **No formal YAML schema** for `sparc.config.yaml`: the parser accepts the subset we use but isn't formally validated. JSON schema is a v0.3.0 story.

## Install / upgrade

```bash
# Fresh install
git clone https://github.com/jb-bz/sparqr.git
cd sparqr
./setup.sh

# Upgrade from v0.1.0
cd sparqr
git pull
./setup.sh   # re-runs the importer; safe to re-run
```
