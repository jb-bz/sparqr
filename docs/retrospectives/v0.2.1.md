# Retrospective: v0.2.1

## What we said we'd do

v0.2.1 was scoped as "make v0.2.0 actually work against real Hermes." Five stories:

1. **Story 1** (5 pts): Fix `lib/kanban.sh` to match real Hermes CLI verbs (which were never tested in v0.2.0)
2. **Story 2** (3 pts): Record real integration sessions
3. **Story 3** (1 pt): Update `TESTED_AGAINST` and uncomment CI workflow
4. **Story 4** (1 pt): OrbStack/Docker runtime selection in `setup.sh`
5. **Story 5** (0-2 pts): Real-Hermes bug fixes discovered during recording

Estimated total: 12 pts (with story 5 sized at 1 pt midpoint of its range).

## What we actually shipped

All 5 stories shipped, plus critical bugs found along the way:

- ✅ **Story 1**: `lib/kanban.sh` rewritten for real Hermes v0.17.0 verb surface (`promote`/`complete`/`block`/`archive`/`claim` for status; positional title for `create`). Output parser updated for `t_<id>` format.
- ✅ **Story 1.5 (unplanned, blocking)**: Fixed the **double-source guard bug** in 7 lib files plus the record-replay harness. Bash functions don't survive across `exec` boundaries; the sentinel-var guards were silently breaking every `sparc <subcommand>` invocation in production. This bug had been latent since v0.2.0.
- ✅ **Story 2**: 3 integration tests with real recorded sessions:
  - `test_setup_against_hermes.sh` — board init + DAG creation (4 assertions)
  - `test_single_stage_run.sh` — state transitions (claim/complete) (2 assertions)
  - `test_two_stage_pipeline.sh` — parent → child DAG with completion (5 assertions)
- ✅ **Story 3**: CI workflow step uncommented with graceful degradation (runs replay-mode test, attempts re-recording if Docker available, skips cleanly otherwise)
- ✅ **Story 4**: `setup.sh` step 5/7 added — asks docker/orbstack/none, recommends based on detection, exports `SPARC_RUNTIME`
- ✅ **Story 5**: Fixed `[UREFINEMENT]` → `[REFINEMENT]` stage-prefix bug (bash 3.2 sed `\U&` incompatibility; replaced with awk)

Total: 5 stories + 1 unplanned blocker fix. **11 integration test assertions + 237 unit tests pass.**

The 3-pt variance on story 1 is justified — we discovered and fixed a production-blocking bug that the unit tests had been masking. That's exactly what v0.2.1 was supposed to catch.

## What surprised us

1. **The double-source guard was silently breaking v0.2.0 in production.** Every `lib/*.sh` had a sentinel-var guard that bailed out on re-source. Bash functions don't survive across `exec` boundaries, but env vars do. So when `bin/sparc` sourced `lib/kanban.sh` (setting `SPARC_KANBAN_LOADED=1`) and then `exec`'d `bin/sparc-init`, the child process saw the env var but had no function definitions. Result: **every dispatched subcommand failed with "command not found" errors in production.** The 237 unit tests passed because they source libs directly, not through the `bin/sparc` dispatcher.

2. **The record-replay harness had 6 bugs in 200 lines.** Same root cause class (state doesn't survive across `exec` boundaries) plus bash-interpolation-into-python-code issues. The harness was written but never end-to-end tested until this release.

3. **`sed 's/^./\U&/'` doesn't work in bash 3.2.** This is a real shell-version gotcha — GNU sed's `\U&` extension is supported in modern GNU sed but the macOS system bash 3.2 ships an older sed that produces literal `\U` instead. We hit it because v0.2.1 actually exercised the kanban wrapper against real Hermes, where the bug surfaced as `[UREFINEMENT]` in the recorded output.

4. **Mock-based testing cannot catch bugs that exist in both the code and the mock.** v0.2.0's mocks were wrong in the same way the code was wrong — both assumed non-existent verb shapes. The unit tests couldn't catch the mismatch because the tests used the same mocks.

5. **The user was right.** This was the second time in this session they said "be sure to test against an actual hermes instance." Both times it surfaced bugs that mocked tests couldn't. This pattern is documented prominently so future sessions don't repeat it.

## What we'd do differently

1. **End-to-end smoke tests are non-negotiable.** The integration test framework existed in v0.2.0 but was never run against real Hermes. v0.2.1 made this mandatory. For v0.3.0, every PR that touches `lib/kanban.sh` MUST run the integration test in RECORD mode against a real Hermes.

2. **The integration test framework needs a "no LLM" mode documented prominently.** This was a real question during this release: should integration tests call real LLMs? The answer is no, for reasons documented in `test_single_stage_run.sh`. This decision needs to be visible to anyone modifying the tests.

3. **Document bugs at every step, even mid-fix.** The WIP status check (`docs/retrospectives/v0.2.1-WIP.md`) was invaluable for not losing track of what was done across two sessions. For v0.3.0, WIP files should be the default for any multi-session work.

4. **The double-source guard pattern is an anti-pattern.** Removed from 7 lib files plus the record-replay harness. Future code should not add it back. The right pattern is "re-sourcing is idempotent for these files" — functions and env vars only, no side effects. If we ever need init code that shouldn't run twice, guard the specific block, not the whole file.

5. **Commit at each real progress point.** This release made 8 commits in this session: guard fix, harness fix, test infrastructure, story 5 fix, story 4, story 3, story 2. Each was independently verifiable. The previous session made commits without verifying they worked — this session fixed that.

## What this means for v0.3.0

- The integration test framework now actually works. v0.3.0 can build on it.
- The lib/ guard pattern is gone; future code shouldn't reintroduce it.
- OrbStack/Docker runtime choice is wired through `setup.sh` and the CI workflow. v0.3.0 features that need a container can rely on this.
- v0.2.1 was supposed to be a 12-pt patch. It became a 14-pt release that fixed a production-blocking bug. **The cost of testing against real infrastructure is high upfront but catches bugs mocks can't.**
