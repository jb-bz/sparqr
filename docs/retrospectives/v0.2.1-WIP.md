## Honest assessment

This is the second time we've tried to record real-Hermes sessions. The first attempt:
- Made real code changes (the verb fixes)
- Did not verify they worked
- Did not record any sessions
- Did not document the partial state

We are now doing it again and hitting the same kind of problems: smoke tests that don't complete, environments that don't get cleaned up, work that doesn't get verified.

**The pattern to break:** we keep treating "the code is written" as "the work is done." It's not. For v0.2.1, every code change needs:
1. A smoke test that proves it works against real Hermes (not just mocks)
2. A documented session recording for the integration test framework
3. An honest pass/fail in this document

If we can't get past step 1 for the verb fix, we should revert to mocks-only and document that the package "works as designed but only against hypothetical Hermes, real verification deferred to v0.2.2."
